1. 데이터 파일 - 물리적 디스크에는 데이터베이스가 여러 데이터 파일들로 구성되어 있고, 각각 데이터 파일은 여러 개의 블록(단위)으로 구성
2. 로그 파일 - 캐시에 저장된 변경 사항이 기록되지 않은 상태에서 장애 발생으로 데이터 파일로 기록하지 못한 정보가 없어진 경우, 복구하기 위해서 사용하는 파일
3. 컨트롤 파일 - 대량의 정렬이나 해시 작업을 수행하다가 메모리 공간이 부족해지면 중간에 결과 집합을 잠시 저장하는 용도로 사용되는 파일
4. 파라미터 파일 - 공유 메모리 영역의 할당 범위, 컨트롤 파일의 경로와 데이터베이스의 환경 설정 ㅡㅇ 관련된 모든 정보를 담고 있는 파일

컨트롤 파일은 데이터베이스에서 사용할 모든 파일들의 절대 경로, 파일 크기 등의 정보를 저장하는 용도이다.

p.4-142 14번

다음 중 아래에서 설명하는 것은 무엇에 대한 설명인가?

```
각 프로세스가 많이 참조하는 페이지들의 집합을 주기억장치 공간에 계속 상주하게 하여 
빈번한 페이지 교체 현상을 줄이고자 하는 기법
```

1. 공간 구역성
2. 워킹 세트
3. PFF
4. 스레싱

정답 2번

공간 구역성은 공간 지역성으로 메모리 참조 현상에 대한 것이다.

PFF는 Page-Fault Frequency로 페이징 방법에서의 페이지 부재율의 상한과 하한을 정하고 페이지 부재 비율에 따라 페이지 프레임 개수를 조절하는 단점 보완 기법이다.

스레싱은 프로세스가 지속적으로 페이지 부재가 발생하여 페이지 교체 시간이 더 많아지는 현상을 일컫는 것으로써 페이징 기법의 문제점을 나타내는 말이다.
